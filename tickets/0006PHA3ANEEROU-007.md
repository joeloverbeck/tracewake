## Problem

The no-human day is a wait-only scheduler scaffold (audit **D-01 / F-01**, the failing capstone), and decision traces/stuck diagnostics are seeded strings, not generated by real choices (**D-07 / F-07**, write side). `run_no_human_day` (`crates/tracewake-core/src/scheduler.rs:275`) iterates windows/actors then calls `build_agent_wait_proposal` (`scheduler.rs:410`), which passes empty needs, `active_intention: None`, `routine_window_goal: None`, hard-coded actor-known strings, and asks the planner for `WaitWithReason`. `AgentState` stores `decision_traces`/`stuck_diagnostics` as `BTreeMap<_, String>`. This ticket replaces the wait-only runner with the autonomous loop spec §5.2 requires, generating structured traces/diagnostics, and handles planner/action mismatch per §5.8.

## What to Change

### 1. Autonomous decision loop

Replace `run_no_human_day`'s wait-only proposal building with a per-actor/window loop performing the §5.2 sequence end-to-end through the shared pipeline, reading live state from dependencies 001–006.

### 2. Structured traces/diagnostics

Change `AgentState` trace/stuck storage to structured records; the loop emits them covering candidate generation, condition resolution, chosen method, action proposal, validation result, fallback/replan/abandonment, and hidden-truth audit.

### 3. Planner/action mismatch handling

A planner action the pipeline rejects is recorded in the trace; the actor replans / continues fallback / waits with reason / fails with diagnostic — never silently idles. Planner budget exhaustion and no-valid-sequence are typed failures.
