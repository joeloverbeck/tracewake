2. INV-018 (replay reconstruction) -> witness asserting `rebuild_projection` selects the reconstructed-final-frontier `sim_tick` only when a `TimeAdvanced` marker is present, and the non-marker branch otherwise (kills `== → !=`).

## What to Change

### 1. `validate_time_advanced` witnesses

Add tests: (a) `resulting_tick == prior_tick` with a fresh frontier → must be accepted (no `BackwardTimeAdvanced`), kills `< → <=`; (b) a case where only `prior_tick < current_frontier` holds and a case where only `resulting_tick <= current_frontier` holds → each expects `DuplicateTimeAdvanced`, killing `|| → &&`.

### 2. `rebuild_projection` marker-branch witness

Add a test asserting the final `ChecksumContext.sim_tick` differs between a log with a `TimeAdvanced` marker (uses `reconstructed_final_frontier`) and one without — so flipping `==`→`!=` on the marker check is observably wrong.

### 3. `is_duplicate_need_tick_candidate` witness

Add a test with a candidate need-tick and a prior `NeedDeltaApplied`/`tick_delta` event at the same `sim_tick` (→ duplicate) vs a different `sim_tick` (→ not duplicate), so `==`→`!=` flips the verdict.
