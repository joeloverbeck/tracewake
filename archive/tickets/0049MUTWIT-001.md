## Problem

The `mutation in-diff (lock layer)` CI gate fails on PR #61 ("Implemented 0048"):
`origin/main...HEAD` changed guarded-layer lines in `scheduler.rs`, and the gate
found surviving mutants there that no test kills (`.cargo/mutants-baseline-misses.txt`
is empty — zero-floor policy). The 0048 capstone (`0048FOUCONHAR-008`) ran a
*narrower* focused campaign (`--no-config`, narrow `-F`) that excluded these
functions, so they were never witnessed. This ticket adds the killing witnesses for
the scheduler surface so the gate goes green without weakening it or growing the
baseline.

Surviving mutants on `scheduler.rs` (from run 28045584094):

1. `latest_current_place_perception_event_id` — `== → !=`, `&& → ||`
2. `latest_need_event_id` — `== → !=`, `|| → &&`
3. `actor_has_open_body_exclusive_at` — `== → !=`, `&& → ||`, `> → <`, `> → ==`, `> → >=`
4. `no_human::append_decision_trace_after_proposal` — `== → !=`
5. `DeterministicScheduler::transact_world_one_tick` — `== → !=` (the two
   `result.report.status == ReportStatus::Rejected` predicates at ~`516` and ~`625`)

## What to Change

### 1. Witnesses for the event-finder predicates

For `latest_current_place_perception_event_id`, `latest_need_event_id`, and `no_human::append_decision_trace_after_proposal`: add tests with a log containing near-miss events (right kind/wrong actor, right actor/wrong tick, right tick/wrong place, and — for `latest_need_event_id` — the `actor_id`-field vs `payload_value("actor_id")` disjunction) so that flipping `==`→`!=` or `&&`/`||` selects a different event id (or `None`) and the assertion fails. Assert on the exact returned `EventId`.

### 2. Witnesses for `actor_has_open_body_exclusive_at`

Add tests covering: a non-body-exclusive duration (kills `&&`→`||` and `==`→`!=` on the `body_exclusive == "true"` check), and completion exactly at `tick` vs strictly after `tick` (kills `> → ==`, `> → >=`, `> → <` on `completion > tick`). Assert the boolean result flips at the boundary.

### 3. Witnesses for `transact_world_one_tick` rejection rollback

Exercise a controlled `ProposalOrigin::Human` proposal that is rejected at both the pre-marker (`~516`) and post-marker (`~625`) predicate sites; assert the run rolls back (physical/agent/epistemic/log/frontier unchanged) so that `==`→`!=` (treating a rejected report as accepted / skipping rollback) is observably wrong.
