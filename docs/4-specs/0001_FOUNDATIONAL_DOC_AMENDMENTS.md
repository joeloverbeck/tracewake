# Spec 0001 Foundational Doc Amendments

No replacement foundational documents are recommended.

## Decision

The foundation, architecture, execution, and reference documents at target commit `3b45d7dde9811f1a0ddbe9497f3d9e0c3743d74e` already establish the controlling doctrine needed for Spec 0001:

- foundation doctrine rejects player privilege, hidden directors, quest ontology, telepathy, abstract property where physical custody is required, and LLM authority;
- architecture contracts already define action proposal/adjudication boundaries, event-log authority, projections, structured speech, claims/beliefs/traces, institutions, ordinary-life simulation, possession/view-model separation, and validation expectations;
- execution docs already designate Phase 0 as the paper ontology and fixture-contract phase before Phase 1 kernel work;
- reference docs already provide exact-source and retread-risk guardrails.

Spec 0001 can resolve the remaining ambiguity by adding concrete Phase 0 contracts under `docs/4-specs/` without rewriting higher-authority documents.

## Why no replacements are included

No replacement document is needed because no analyzed source document contradicts a higher authority in a way that blocks Phase 0. The remaining gap is not doctrine; it is specificity. Spec 0001 supplies that specificity through reviewable contracts for actors, places, containers, physical value tokens, beliefs, speech, institutions, events, propositions, fixtures, view models, validation mapping, and Phase 1 entry.

No `docs-replacements/` directory is included in this package.

## Future amendment trigger

A future replacement or addition should be considered only if later work discovers that the existing docs, even with Spec 0001 accepted, invite one of these failures:

- quest/objective ontology;
- player privilege or action-pipeline bypass;
- epistemic leakage;
- institutions reading truth instead of claims/evidence;
- live LLM text mutating state;
- abstract balances replacing physical custody;
- implementation-ticket thinking inside doctrine;
- retreading Phase 0 instead of moving to Phase 1.
