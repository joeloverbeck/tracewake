# Spec 0001 Research Notes

These notes summarize the focused external research used to pressure-test Spec 0001. They are not a bibliography dump and do not override Tracewake doctrine casually.

## Research summary table

| Topic | Sources | Finding | Tracewake implication | Existing docs changed? | Role in Spec 0001 |
|---|---|---|---|---|---|
| Event sourcing, replay, CQRS, projections | Martin Fowler, “Event Sourcing”; Microsoft, “Event Sourcing pattern”; Microsoft, “CQRS pattern” | Event sourcing treats state changes as an append-only event sequence and can rebuild application state by replaying events. CQRS commonly separates write-side events from read-side projections/materialized views. | Phase 0 should name primitive event families, projection effects, replay requirements, save-package inputs, and forbidden causes before Phase 1 kernel work. | No. Existing docs already require event log authority, replay, and projections. | Pressure-tested event vocabulary and replay/rebuild fixture. |
| Event schema/version risk | Microsoft Event Sourcing pattern | Event sourcing provides audit/replay benefits but introduces complexity, including event-structure evolution and replay/projector concerns. | Spec 0001 avoids final serialization syntax and focuses on stable semantic event families and referenced causes. | No. | Reinforces not locking final file syntax in Phase 0. |
| Deterministic simulation and seeded randomness | Deffuant et al., “Taming Randomness in Agent-Based Simulation”; Scientific Python Blog, “Best Practices for Using NumPy's Random Number Generators” | Pseudorandom generation is deterministic under seed, but global or centralized random streams can damage reproducibility when model structure changes. Independent streams can preserve auditability better. | Phase 0 event families include seed references and no-human/replay markers, but final RNG strategy remains Phase 1+ architecture detail. | No. | Pressure-tested replay/rebuild and no-human fixtures. |
| BDI agents, durable intentions, HTN routines, bounded planning, GOAP/STRIPS | Springer, “The Belief-Desire-Intention Software Model”; IBM Research, SHOP2 HTN planner; Jeff Orkin / F.E.A.R. GOAP resources | BDI is a standard practical-reasoning agent model; HTN routines support structured procedural behavior; GOAP-style planning provides bounded action selection when goals/preconditions/effects are explicit. | Phase 0 should define action preconditions, belief preconditions, intention/plan event hooks, and routine continuation/abandonment without implementing full cognition. | No. | Pressure-tested action vocabulary and deferred routine semantics. |
| Smart objects and affordance-based interaction | Contextual-affordance work in intelligent virtual characters; Gibson-derived affordance framing in HCI literature | Objects and places should advertise actions based on actor state, permissions, and context, not fixed quest verbs. | Phase 0 places, doors, containers, and items define affordances, visibility, and failure cases. | No. | Supports object/container/door affordance tables. |
| Talk of the Town-style character knowledge | Ryan et al., “Talk of the Town: Character Dialogue…” | Talk of the Town centers character knowledge propagation, routines, social networks, and partial/mistaken information as player-facing material. | Spec 0001 treats speech, reports, rumor, observation, memory, and belief as structured, source-bound artifacts. | No. | Supports actor-filtered belief and speech contracts. |
| Prom Week / Comme il Faut-style social state and rules | McCoy et al., Prom Week / Comme il Faut | Prom Week demonstrates heavy rule-based social state and social-rule reasoning. | Tracewake should keep social rules structured and explainable, but Phase 0 should avoid a large rule-system commitment. | No. | Supports minimal norms and typed claims rather than freeform drama logic. |
| Neighborly/community-scale simulation | Lawley & MacLellan, “Neighborly”; Neighborly repository | Neighborly demonstrates settlement-scale social simulation using traits, relationships, occupations, life events, and level-of-detail abstractions. | Spec 0001 keeps the first village small, but defines enough ordinary-life roles, relationships, and pressures to support later no-human life. | No. | Pressure-tests actor roster and LOD boundaries. |
| Agent-based social simulation validation | David, “Methods That Support the Validation of Agent-Based Models” | ABM validation benefits from multiple methods: visualization, docking, empirical checks, sampling, causal analysis, role-playing, and more. | Spec 0001 maps fixtures to future golden tests, property tests, replay tests, TUI/view-model tests, no-human tests, content validation, and leakage tests. | No. | Supports validation section. |
| Normative multi-agent systems and institutional facts | Boella & van der Torre, “Regulative and Constitutive Norms in Normative Multiagent Systems” | Norms can regulate behavior through obligations/prohibitions/permissions and can constitute institutional facts; violations are possible and meaningful. | Spec 0001 keeps violation, detection, suspicion, report, record, proof, sanction, notice, and payment separate. Records are claims, not truth. | No. | Supports institution/norm/record contracts. |
| Dynamic level-of-detail and multi-resolution simulation | Neighborly paper; Tracewake foundation and architecture LOD docs | Community simulation can summarize distant or low-relevance activity while preserving high-detail causal explanation where needed. | Spec 0001 confines Phase 0 to a small high-detail village and leaves larger LOD promotion/demotion to later specs. | No. | Supports small-village scope. |
| Emergent narrative and story-sifting | Ryan et al., “Winnow: A Domain-Specific Language for Incremental Story Sifting” | Story sifting can curate generated events without directly modifying the event-generating simulation. | Spec 0001 allows future incident/leads/story-sifting projections but forbids hidden drama directors and forced theft/discovery/closure. | No. | Supports no-scripting checklist. |
| Game precedents and counterexamples | Shadows of Doubt public descriptions; Dwarf Fortress/Legends precedents; RimWorld storytellers; Left 4 Dead AI Director literature | Persistent simulated spaces and history/debug views are useful precedents. Drama/story directors that inject events for pacing are counterexamples for Tracewake’s first proof. | Spec 0001 adopts persistent-world/debug lessons and rejects player-conditioned events, quest flags, rewards, and hidden culprit pointers. | No. | Supports explicit non-goals and no-scripting review. |
| LLM social simulation validation risks | Larooij & Törnberg, “Beyond Chatbots…”; Generative Agents; Concordia | LLM agents can produce rich social behavior, but black-box behavior, cultural bias, stochasticity, and face-validity problems make validation risky. | Spec 0001 keeps LLMs behind structured speech/rendering/parsing boundaries and forbids freeform LLM speech or state mutation. | No. | Supports speech/report contract and LLM non-goal. |
