# Research Decisions and Source Notes

## Status

This document records stable execution-specific research decisions and source-handling policy.

It is not a literature review, implementation ticket list, migration note, session transcript, exact-commit audit ledger, or connector history. Exact repository fetch ledgers belong in audit outputs, not in committed execution doctrine.

## Source-handling policy

Execution documents may record stable doctrine and durable source lessons. They must not preserve one-off review mechanics as product authority.

Do not commit:

- target commit IDs from a review session;
- exact raw URL fetch ledgers;
- branch names as authority;
- default-branch assertions;
- connector namespace labels as evidence;
- uploaded temporary source-material lists as enduring doctrine;
- stale migration notes;
- claims that a reviewed commit is current `main` unless independently verified by the review task.

When an audit requires exact-commit discipline, the audit artifact must carry the evidence ledger for that audit. Product docs should say what the product requires, not what one review session fetched.

## Stable execution research decisions

### Talk of the Town

Useful lesson: small settlements become interesting when characters have situated knowledge, memory, social relationships, observation, conversation, forgetting, and lies.

Execution consequence: Tracewake's first proof must include subjective beliefs, stale or false beliefs, witness uncertainty, testimony, gossip, and source attribution. It must not use omniscient culprit knowledge.

### Neighborly and community-scale simulation

Useful lesson: community-scale simulation can generate rich small-town social history from homes, businesses, roles, relationships, and life events.

Execution consequence: Tracewake should start with a small village-scale fixture and ordinary-life routines. It should not jump to regional adventure systems before property, belief, records, and no-human local life are proven.

### BDI, HTN, GOAP, and inspectable planning

Useful lesson: transparent belief/desire/intention structures, task decomposition, and bounded planning can produce resourceful behavior while remaining debuggable.

Execution consequence: first-proof agents should be symbolic and inspectable before any generative layer is considered. The missing-property chain should be explainable as beliefs, intentions, routines, affordances, and plan updates, not hidden LLM vibes.

### Agent-based modelling validation

Useful lesson: validation must be tied to the intended purpose of the model. A simulation that looks plausible can still fail its research/product purpose.

Execution consequence: Tracewake phase gates must test the constitutional identity directly: replay, provenance, actor-grounded views, no-human operation, subjective beliefs, physical custody, fallible institutions, and no-director behavior.

### Dwarf Fortress-style history generation

Useful lesson: historical/prehistory generation is a powerful long-term inspiration when past events leave persistent consequences.

Execution consequence: Tracewake may eventually simulate months or years before play begins, but only if promoted actors and incidents retain enough ancestry to explain current beliefs, injuries, debts, rumors, relationships, goods, and records.

### RimWorld-style AI storyteller as counterexample

Useful lesson: a storyteller/director that creates or paces events can produce enjoyable drama, but it violates Tracewake's foundation.

Execution consequence: no execution phase may introduce a hidden director to spawn, pace, repair, conclude, or reward events for dramatic effect.

### Generative Agents and LLM social simulation

Useful lesson: LLM-based memory, reflection, and planning can produce striking social believability.

Execution consequence: Tracewake must keep LLMs behind structured speech/rendering/parsing boundaries until deterministic mocks, validation, replay, and non-authority rules are already proven. Live LLMs inside agents are not first-proof execution work.

### Story sifting and curationist narrative

Useful lesson: story sifting can extract and present narrative shape from event chronicles without controlling the event generator.

Execution consequence: incidents, leads, summaries, and retrospectives may be projections over already-causal events. They may not mutate state, select culprits, spawn clues, guarantee closure, or pace the world.

### Gossip, rumor, testimony, trust, and reputation

Useful lesson: belief propagation is shaped by source, social network, trust, role, confidence, and repeated transmission.

Execution consequence: first-proof speech/report mechanics should transmit structured claims with source attribution and trust/confidence effects. Speech should not copy truth; it should alter beliefs through deterministic rules.

### Norms, records, and institutions

Useful lesson: institutions can create procedural facts, obligations, permissions, sanctions, records, and roles. Procedural validity is not the same thing as truth or justice.

Execution consequence: the first local authority must act from institutional knowledge and procedure, not ground truth. Records may be partial, stale, false, missing, biased, or procedurally valid but morally wrong.

## Execution research stance

Research sources are pressure tests, not blueprints.

Tracewake should borrow:

- actor-specific knowledge propagation;
- small-community social simulation;
- inspectable cognition;
- explicit validation;
- history with causal ancestry;
- retrospective story sifting;
- structured speech and belief transmission;
- formal institutional/normative distinctions.

Tracewake should reject:

- quest scripting;
- hidden directors;
- omniscient UI mystery solving;
- protagonist metaphysics;
- opaque LLM state authority;
- story managers that alter outcomes;
- abstract scale that cannot explain ancestry;
- demos that look alive but cannot be replayed or audited.

## Execution consequence

The first proof remains narrow:

```text
physical expected property
-> absence discovered
-> belief contradiction
-> search/question/report
-> partial testimony or rumor
-> institutional record
-> possible wrong suspicion
-> possession parity
-> no-human operation
-> replayable provenance
```

Any execution plan that makes this chain secondary is drifting.
